Cascading Style Sheets (CSS)

History and Evolution of CSS

Cascading Style Sheets (CSS) is one of two core technologies in web development, the other being HTML. CSS is a style sheet language that is used to style the aesthetics and layout of a web page. CSS allows the presentation of a web page to adapt to various media types such as desktops, mobile devices and prints. The language was first proposed by Hakon Wium Lie in 1994 to complement the initial version of HTML, developed by his colleague, Tim Berners-Lee. In 1996, computer scientist Bert Bos join the World Web Consortium (W3C) to work on CSS, and is regarded as the co-creator of CSS, along with Hakom Wium Lie.

CSS 1 (1996)

This was the first specification of CSS that became an official recommendation of the W3C. This version of CSS supported the following:

  • Font properties
  • Text and background colours
  • Text attributes
  • Element alignment, margin, border and padding attributes
  • Unique identification and attribute group classifications.

CSS 2 (1998)

This specification of CSS was developed and published by the W3C. It was an enhancement of CSS1 and included notable new capabilities such as:

  • Positioning of elements - Absolute, Relative and Fixed
  • Support for different media types - Aural, Braille, Handheld etc.
  • Wider range of selectors
  • New pseudoclasses and pseudoelements

CSS 3 (1999)

Unlike previous versions of CSS which were based on large single specifications, CSS3 was divided into multiple "modules", with each module extending features or adding new capabilities to previously defined features. Due to modularisation, different modules have varying stability and statuses (Working Drafts/Candidate Recommendation/Recommendation). CSS3 modules are still being updated, with the 'Selectors Module 3' recently becoming a 'Recommendation' in November, 2018. Some important features released in CSS3:

  • Support for web fonts
  • Support for 2D/3D animations and transitions (without Flash/JavaScript)
  • Support for RGBA, HSLA, HSL and Gradient colours

Notable CSS3 Modules:

  • Media Queries
  • CSS Fonts Module Level 3
  • CSS Color Module Level 3

Formally recommended in 2012, the 'Media Queries' module is considered as one of the most revolutionary modules from the CSS3 release and is responsible for the responsive designs of modern day web pages.This module analyses the capabilities of mobile devices used and defines tailored style sheets for the various devices (laptops, mobile phones, smartwatches etc.) we use in the modern world.

Levels of CSS

When styling a webpage using CSS, there are 5 important levels that a web developer should consider. Listed in descending order of cascading priority, they are :

  1. User-defined styling
  2. Inline CSS
  3. Internal CSS
  4. External CSS
  5. Browser-default styling

User-defined style sheets are mainly used by users who require added accessibility, and write their own CSS to customise and override a web page's default style. On the other hand, browser-default styles are the different rendering styles of a particular browser. Each browser may have different styling properties for each HTML element. Purpose and Benefits of CSS

Apart from visual enhancements, there are many significant reasons to justify the use of CSS in web pages. Below are some of the benefits of using CSS.

Consistency In Design

Design consistency is one of the most important considerations in web development. The layout of a website should be consistent across all its pages as it reflects on the overall outlook of the website. Using CSS allows the design for a particular element to be repeated within a page and even across multiple pages. This also smoothens the maintenance of the website, as styles only need to be edited once in order to be reflected throughout the entire website, saving the developer a lot of time.

Increased Web Page Performance

Storing all CSS code on an external sheet, results in a lightweight HTML document which in turn helps reduce download times.Through the implementation of CSS, the code is generally cached in the browser and does not need to be downloaded again to load the other pages, reducing the network bandwidth usage.

Search Engine Optimisation

As mentioned above, the implementation of CSS results in a lightweight HTML document as it contains less code. This simpler structure improves readability not only for the developer, but also for search engine algorithms. A lightweight documents allows for algorithms to search through the code at a much quicker rate. That being said, it is more likely for a lightweight webpage to rank higher in search results.

CSS Selectors

CSS Selectors are used to define the HTML elements to which a set of CSS rules apply.They can categorised into the following groups:

  • Simple - selects elements based on element name, id and class etc.
  • Combinator - selects elements based on relationships (descendent, child, sibling etc.)
  • Pseudo-class - selects elements by state
  • Pseudo-elements - selects elements by specific parts of elements
  • Attribute - Selects elements based on attribute or attribute value

Pseudo-element Selectors

Psuedo-element Selectors are used when styling specific parts of an element. There are six types of pseudo-elements and they are:

  • ::after - inserts content after each element
  • ::before - inserts content before each element
  • ::first-letter - selects the first letter of each element
  • ::first-line - selects the first line of each element
  • ::selection - selects the portion of an element that is highlighted by the user.
